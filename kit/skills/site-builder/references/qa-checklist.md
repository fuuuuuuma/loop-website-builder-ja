# QAチェックリスト — サブエージェントが各観点を0-100で採点する基準

`/goal` ループで、**3体以上のサブエージェントが各観点100点になるまで**回す（`prompts/goal-loop.md`）。
**自己採点しない。** 各チェッカーは下の項目を実機で確認し、0-100点＋具体的な修正指示を返す。
Anthropic の記事どおり「**数えられるチェックほど収束が速い**」ので、まず定量項目を潰す。

---

## ① 機能チェッカー（「側だけ」を許さない）
- [ ] **リンク切れ 0**：全 `<a>` の遷移先が実在する（別ページ／外部URL／`#anchor`）。`href="#"` が残っていない
- [ ] **動かないボタン 0**：全 `<button>` が押して反応する（モーダル開閉・タブ・送信・スクロール）
- [ ] **フォーム送信が成立**：送信先が配線済み（メール／外部フォーム等）。未定は「準備中モーダル」で閉じ/背景/Esc が効く
- [ ] **戻れる**：モーダル/別ページから元に戻れる。ハンバーガー・アコーディオンが開閉する
- [ ] **コンソールエラー 0**（JSエラー・404 が出ていない）
- [ ] スムーススクロール・アンカーが正しい位置に飛ぶ
- 100点の条件: 上のすべてが実機（ブラウザ）で確認できる

## ② デザインチェッカー
- [ ] **DESIGN.md 準拠**：色は選んだ `:root` トークンのみ。トークン外の色使用 0
- [ ] **参考サイトのトンマナに一致**（余白・階層・フォント・雰囲気）。差分レビューで違和感なし
- [ ] 視線導線が自然（ヒーロー→価値→CTA）。1画面1メッセージ
- [ ] コントラスト AA（本文4.5:1／大見出し3:1）。accent地に白文字を使っていない
- [ ] 画像/図の質が揃っている（解像度・トリミング・余白）
- 100点の条件: DESIGN.md と参考サイトに照らして、直す点が無い

## ③ モバイル&速度チェッカー
- [ ] **390 / 430 / 768 / 1024 / PC** で横スクロール 0・崩れ 0
- [ ] タップ領域 44px 以上・文字が小さすぎない（本文16px目安）
- [ ] **Lighthouse ≥ 90**（できれば Performance/Best Practices/SEO/A11y すべて）
- [ ] **LCP < 2.5s ／ CLS < 0.1 ／ INP 良好**。画像は適切サイズ・`loading="lazy"`・幅高さ指定
- [ ] `prefers-reduced-motion` でアニメが止まる
- 100点の条件: 各幅で崩れ0＋速度指標クリア

## ④（任意）コピー&法規制チェッカー
- [ ] 誇張・断定保証が無い（必ず/絶対/確実/裏技/神/ヤバい/衝撃/100%）
- [ ] 数字・実績は出典があり事実（景表法）。医療/健康表現は薬機法に配慮
- [ ] 特商法表記・プライバシーポリシー・運営者情報（必要なら）
- [ ] ステマ規制（広告・PR の明示）
- 100点の条件: 表現・表記に法的リスクが無い

---

## スコープ（全チェッカー共通の減点）
- `site-spec.md` に無いページ・機能・コピーを足していたら減点（Simple is the best）
- プレースホルダ・スタブ・「あとで」を完了として報告していたら 0 点扱い
