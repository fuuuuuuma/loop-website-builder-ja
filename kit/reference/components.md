# コンポーネントの種類を知る — 何を並べてサイトを作るか

サイトは「ページ全体のセクション」と「その中で使う UI 部品」の2階層でできている。
**先に部品名を知っておくと、AI に「ここは○○を使って」と正確に頼める**（＝ループが速くなる）。
UI 部品の分類は Google の [Material 3 — Components](https://m3.material.io/components) を参照（名称は一般的なWeb用語に合わせて表記）。

---

## A. ページのセクション部品（上から下に並べる大きな塊）

| 部品 | 役割 | 使いどころ |
|---|---|---|
| **ナビバー / ヘッダー** | 上部の案内。ロゴ＋メニュー＋CTA | 全ページ共通 |
| **ヒーロー** | 最初の大画面。一言＋CTA＋画像 | トップ最上部 |
| **ロゴクラウド / 実績バー** | 取引先・掲載・数字で信頼 | ヒーロー直下 |
| **特徴グリッド** | 価値をカードで3〜6個 | 中盤 |
| **ベントーグリッド** | 大小のタイルで魅力を一望 | 特徴の見せ方 |
| **ステップ / フロー** | 使い方を①→②→③ | 使い方説明 |
| **比較表 / before-after** | 自分 vs 従来 | 差別化 |
| **料金テーブル** | プランを3カード | 販売系 |
| **お客様の声** | 引用・顔写真・星 | 信頼補強 |
| **FAQ（アコーディオン）** | 不安をつぶす | 終盤 |
| **CTAセクション** | もう一度背中を押す | フッター直前 |
| **フッター** | リンク・会社情報・規約 | 最下部 |

## B. UI 部品（セクションの中で使う小さな部品・Material 3 の分類）

**アクション系（Actions）**
- ボタン（一次/二次/テキスト）、FAB（フローティング操作ボタン）、アイコンボタン、セグメント/トグルボタン

**ナビゲーション系（Navigation）**
- トップ/ボトム アプリバー、タブ、ナビゲーションバー、ナビゲーションレール、ナビゲーションドロワー、
  パンくず（breadcrumb）、ページネーション、メニュー（ドロップダウン）

**入力・選択系（Text inputs / Selection）**
- テキストフィールド、検索バー、チェックボックス、ラジオボタン、スイッチ、スライダー、チップ（chips）、
  日付ピッカー、時刻ピッカー、セレクト

**入れ物系（Containment）**
- カード、ダイアログ（モーダル）、ボトムシート、ディバイダー（区切り線）、リスト、カルーセル、ツールチップ

**通知・状態系（Communication）**
- バッジ、スナックバー/トースト、プログレスインジケーター（ローディング）、スケルトン（読込プレースホルダ）

---

## 選び方（迷ったら）
- **要素は足しすぎない**（Simple is the best）。1セクション1目的。
- Material 3 は「部品の名前と役割」を知る辞書として使う。**見た目は `design-systems/` の DESIGN.md** に合わせる
  （M3のマテリアル調をそのまま真似る必要はない）。
- 「この部品、押した先まで動く？」を必ずチェック（→ `site-playbook.md` の「側だけダメ」）。
